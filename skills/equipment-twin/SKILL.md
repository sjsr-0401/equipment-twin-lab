# Equipment Twin Lab Project Skill

이 문서는 Codex와 Claude Cowork가 `Equipment Twin Lab` 프로젝트를 다룰 때 참고할 프로젝트 지식이다.

## 목적

사용자가 장비 SW 엔지니어로서 설명 가능한 대표 프로젝트를 만든다.

핵심은 실제 장비 없이도 아래 요소를 소프트웨어로 모델링하는 것이다.

- 장비 상태 전이
- PLC/IO 신호
- 모션 축
- 센서
- 카메라 검사
- 알람
- 공정 시나리오
- 3D 시뮬레이션

## 설계 원칙

- Unity는 시각화 계층으로 둔다.
- 핵심 장비 로직은 C#/.NET 라이브러리로 분리한다.
- 실제 장비 의존성은 인터페이스 뒤로 숨긴다.
- 가상 장비와 실제 장비가 같은 API를 쓰는 구조를 지향한다.
- 사용자가 이해할 수 없는 복잡한 자동화는 대표 프로젝트 품질을 낮춘다.

## 설명 기준

작업 결과는 항상 아래 관점으로 설명한다.

1. 제조 장비에서 이 기능이 어떤 역할을 하는가
2. 실제 장비가 있다면 어떤 하드웨어와 연결되는가
3. 지금 구현은 시뮬레이션인가, 실제 코드로 확장 가능한 구조인가
4. 테스트가 무엇을 검증하는가
5. 면접에서 어떻게 말할 수 있는가

## 주의

- 실제 장비 안전성을 증명했다고 말하지 않는다.
- 카메라 검사는 데이터/광학/조명/캘리브레이션 영향을 받으므로, 가상 검사의 한계를 함께 설명한다.
- CAD/Blender 모델은 외형이다. 동작 로직과 센서 기준점은 별도로 관리한다.
