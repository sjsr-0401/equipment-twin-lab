# Agent Instructions

이 워크스페이스의 주 목적은 `Equipment Twin Lab` 대표 프로젝트를 만들고, 사용자가 장비 SW 엔지니어로서 제조 공정/장비 제어/검사 자동화 역량을 설명할 수 있게 정리하는 것이다.

## 항상 먼저 읽을 문서

작업을 시작하기 전에 아래 문서를 확인한다.

1. `plan.md`
2. `state/loop-state.md`
3. `state/triage.md`
4. `state/decisions-needed.md`

## 작업 원칙

- 사용자는 4일 근무/4일 휴무 패턴이며, 근무일에는 하루 약 1시간만 쓸 수 있다.
- 결과물은 초보자도 이해할 수 있게 설명한다.
- 코드만 만들지 말고, “무엇을 했는지 / 왜 했는지 / 어떻게 확인하는지 / 사용자가 무엇을 이해해야 하는지”를 남긴다.
- 장비 SW, 제조 공정, PLC/IO, 모션, 카메라 검사, 시뮬레이션 관점에서 실무성이 있는지 계속 점검한다.
- 실제 장비 안전 로직처럼 보이는 내용은 검증 없이 확정 표현하지 않는다.

## 자동화 권한

- 현재는 수동 루프 단계다.
- 자동 병합은 아직 허용하지 않는다.
- 문서, 오탈자, 작업 로그 같은 L0 변경만 자동화 후보로 둔다.
- 코드 변경, 구조 변경, 테스트 정책 변경은 사용자가 확인할 수 있게 요약한다.

## 에이전트 협업 규칙

- Codex와 Claude Cowork가 같은 프로젝트를 다룰 수 있으므로, 결정사항은 대화가 아니라 파일에 남긴다.
- 병렬 작업이 필요할 때는 서로 같은 파일을 동시에 고치지 않도록 작업 범위를 분리한다.
- 작업 완료 후에는 `state/comprehension.md` 또는 `logs/`에 사용자 이해용 요약을 남긴다.
