## 사용자가 이해해야 할 것

이번 작업의 핵심은 코드를 만든 것이 아니라, Codex와 Claude Cowork가 같은 기준으로 일할 수 있게 “기억 장치”를 만든 것이다.

에이전트는 대화가 끊기면 맥락을 잊을 수 있지만, 파일에 남긴 상태와 결정사항은 다음 작업에서 다시 읽을 수 있다.

추가로 첫 코드 MVP를 만들었다.

장비 상태머신은 장비가 현재 어떤 단계에 있는지 관리하는 기본 구조다. 실제 장비에서는 PLC 신호, 모션 완료, 센서 입력, 카메라 검사 완료 같은 이벤트가 상태 전이를 일으킨다.

가상 IO 모델은 실제 PLC 없이 센서 입력과 출력 명령을 분리해서 테스트하는 구조다.

Input은 센서/PLC가 바꾸고 장비 SW가 읽는 값이다. Output은 장비 SW가 쓰고 밸브/램프/부저 같은 장치가 반응하는 값이다.
