## 소프트웨어 아키텍처 설명

```text
사용자 선택 장비/공정
    ↓
Equipment Template
    ↓
Motion Axis / IO / Inspection / Fault Model
    ↓
Scenario Runner 또는 Unity Adapter
```

오늘 추가한 것은 이 중 `Motion Axis`다.

```text
ServoOn
    ↓
StartHome
    ↓
Poll
    ↓
StartMove
    ↓
Poll 또는 CheckTimeout
    ↓
InPosition 또는 Alarmed
```

`MotionAxis`는 실제 서보 드라이버가 아니다. 실제 장비 없이 모션 제어 흐름을 소프트웨어로 검증하기 위한 모델이다.

시간은 `ManualClock`으로 제어한다. 그래서 테스트에서 실제로 3초를 기다리지 않고도 3초 뒤 동작을 바로 검증할 수 있다.

## 사용자가 이해해야 할 개념

모션 축은 장비의 움직이는 부품 하나다.

예:

- X Stage
- Z Lift
- Conveyor
- Robot Arm Axis

각 축은 대체로 아래 흐름을 가진다.

```text
Servo Off → Servo On → Home → Move → InPosition
```

문제가 생기면 알람으로 간다.

```text
Move 중 지정 시간 초과 → MoveTimeout → Alarmed
```
