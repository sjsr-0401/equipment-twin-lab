# Work Logs

매일 작업한 내용을 짧게 남기는 폴더다.

권장 파일명:

- `YYYY-MM-DD.md`

각 로그에는 아래 내용을 남긴다.

1. 오늘 한 일
2. 바뀐 파일
3. 확인한 것
4. 사용자가 이해해야 할 것
5. 다음 작업

## 작업별 기록 템플릿

하루에 여러 Goal을 진행하면 아래 형식으로 Goal별 섹션을 추가한다.

```markdown
## Goal NNN: 작업 이름

### 한 일

-

### 바뀐 파일

-

### 검증

- 로컬:
- GitHub Actions:

### 사용자가 이해해야 할 것

-

### 한계와 다음 작업

-
```

규칙:

- “테스트 통과”라고만 쓰지 말고 몇 개가 통과했는지 쓴다.
- 실제 장비 안전성을 검증한 것처럼 쓰지 않는다.
- Claude Cowork가 리뷰할 수 있게 면접용 설명 포인트를 남긴다.
